# Transformer Explainer 中文模型替换全流程梳理

## 1. 方案目标

- 将原项目英文 GPT-2 替换为中文 GPT-2（如 uer/gpt2-chinese-cluecorpussmall），实现中文输入、推理与可视化。
- 保持前端架构、可视化逻辑和 UI 不变，最小化代码改动。

---

## 2. 环境准备

- Python 3.8+
- Node.js 20+
- PyTorch、transformers、onnx、onnxruntime、huggingface_hub
- VS Code 或其他编辑器
- 6GB+ 磁盘空间

---

## 3. 步骤详解

### Step 1：下载中文 GPT-2 模型

- 推荐模型：uer/gpt2-chinese-cluecorpussmall
- 运行 01.py 脚本，自动下载模型权重和配置到 models/gpt2-chinese-cluecorpussmall/

### Step 2：转换为 ONNX 格式

- 运行 02_convert_onnx.py
- 复用 src/utils/model/model.py 架构，加载中文权重，导出 ONNX 文件到 models/gpt2-chinese-onnx/model.onnx
- 验证 ONNX 推理输出

### Step 3：模型分片

- 运行 03_chunk.py
- 将 model.onnx 分片为 10MB/片，输出到 static/model-v2-chinese/
- 记录分片总数，供前端加载用

### Step 4：生成中文示例数据

- 运行 04_gen_examples.py
- 用 ONNX 推理生成 ex0~ex4.js，覆盖 src/constants/examples/ 下原有英文示例

### Step 5：前端集成与代码修改

- 修改 src/routes/+page.svelte：
  - 加载中文模型分片目录和数量
  - 加载中文 tokenizer
- 修改 src/store/index.ts：
  - 替换 inputTextExample 为中文示例
- 检查 src/utils/textbookPages.ts，必要时同步示例内容

### Step 6：本地测试与验证

- npm run dev 启动前端，检查：
  - 页面加载、模型加载、推理、可视化是否正常
  - 示例数据、token 显示、attention 矩阵等是否正确

---

## 4. 常见问题与风险提示

- ONNX 转换失败：检查 torch/transformers/onnxruntime 版本，或尝试不同导出方式
- Tokenizer 不兼容：优先用 BertTokenizer，确保 vocab.txt 与模型一致
- 性能问题：可用量化 ONNX 或减小分片
- 推理异常：检查权重加载、输入输出 shape

---

## 5. 参考脚本

- 01.py：下载模型
- 02_convert_onnx.py：权重转换与 ONNX 导出
- 03_chunk.py：模型分片
- 04_gen_examples.py：生成示例数据

---

- 全流程无需重构前端，仅需替换模型、分片和示例数据。
- 适合快速迁移、复现和开源说明。
